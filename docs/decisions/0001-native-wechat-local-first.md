# ADR-001：使用原生微信小程序与本地优先存储

## 状态

Accepted

## 日期

2026-07-17

## 背景

项目面向个人长期使用，需要在中国大陆网络环境中稳定运行，同时记录训练、身体、饮食和睡眠等敏感健康数据。第一阶段希望避免服务器成本、账号系统和第三方框架升级负担。

## 决策

- 使用原生微信小程序语法实现移动端产品。
- 使用微信本地存储保存个人状态。
- 网页版本预览器保留为独立工具，用于比较 V5–V7 变化，不承担生产数据存储。
- 所有本地状态通过单一存储服务读写，备份恢复经过严格白名单。

## 备选方案

### 跨端框架

能同时输出多个平台，但当前只有微信小程序一个明确目标。额外构建链、运行时差异和升级成本暂时不值得。

### 第一版即使用云数据库

可以天然跨设备，但会引入账号、权限、隐私说明、费用和运维。项目尚未验证长期使用习惯，先保持本地优先更合适。

## 后果

- 工程可以直接导入微信开发者工具，依赖少、维护简单。
- 换设备需要用户主动导出/恢复备份，暂时没有自动同步。
- 将来增加云同步时，必须保留本地模式并设计清晰的数据迁移方案。
