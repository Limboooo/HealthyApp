# ADR-002：照片营养分析必须经过用户确认

## 状态

Accepted

## 日期

2026-07-17

## 背景

单张照片无法可靠判断实际克重、隐藏食材、烹饪油、糖和酱料。中餐尤其容易因为做法不同产生较大热量误差。照片和饮食记录也属于敏感个人信息。

## 决策

- 产品流程固定为“选择照片—模型估算—用户修正—确认保存”。
- UI 必须显示“估算值”和可信度，不把结果描述为精确测量。
- 第一阶段使用本地模板演示，不上传或保留照片。
- 接入真实模型时，只能由云函数持有 API Key；前端不得包含密钥。
- 上传前说明用途，分析后默认删除原图，并提供调用限额和失败后的手动记录入口。

## 备选方案

### 识别后直接保存

交互更短，但会让明显错误进入长期趋势，削弱用户信任，因此拒绝。

### 在小程序前端直接调用模型

实现简单，但密钥可被提取，且难以执行统一的限额和删除策略，因此拒绝。

## 后果

- 多一步确认，但营养记录更透明、更可修正。
- 真实 AI 接入需要云函数和隐私配置，不能只替换一个前端 URL。
