# 我的哑铃教练 V7 · 原生微信小程序

这是不依赖框架和第三方运行时的原生微信小程序工程。所有训练、身体、饮食、睡眠和打卡数据默认保存在当前设备的微信本地存储中。

## 已实现

- 今日页：A/B 训练轮换、恢复日、健康分、连续打卡和数据摘要。
- 私人教练：按睡眠、蛋白质、饮水和训练完成情况生成本地规则建议。
- 训练跟练：逐组重量、次数、RPE、90 秒休息、训练历史和下次重量建议。
- 打卡成就：月历、当前/最长连续天数、累计天数和成就徽章。
- 健康数据：身高、体重、体脂率、腰围、睡眠、蛋白质和饮水记录。
- 饮食记录：拍照/相册入口、本地餐食模板、份量修正、烹饪方式、营养估算。
- 数据安全：本地备份导出/恢复、恢复数据字段白名单和记录数量限制。

## 导入微信开发者工具

1. 打开微信开发者工具，选择“导入项目”。
2. 项目目录选择本文件所在的 `miniprogram-v7` 文件夹，不要选择里面的 `miniprogram` 子目录。
3. 体验阶段可继续使用 `project.config.json` 中的 `touristappid`。
4. 真机预览或发布前，把 `project.config.json` 的 `appid` 替换为你在微信公众平台取得的小程序 AppID。
5. 点击“编译”；底部应显示“今日、教练、打卡、数据”四个入口。

工程已设置 `miniprogramRoot: "miniprogram/"`，所以开发者工具会自动识别源码目录。

## 真机预览检查

- 今日页能根据日期显示训练 A、训练 B 或恢复日。
- 完成一组训练后出现 90 秒倒计时；结束训练后当天自动点亮。
- 恢复日需要完成散步、拉伸、补水三项任务。
- 身体数据保存后，重新进入页面仍然存在。
- 选择餐食照片时仅在本页预览；保存记录不会保留或上传原图。
- 复制备份后可以从剪贴板恢复，畸形或超大备份会被拒绝。

## 发布前必须配置

1. 在微信公众平台完成小程序主体、名称、类目和管理员配置。
2. 在“用户隐私保护指引”中如实说明相机/相册用于选择餐食图片、剪贴板用于用户主动备份恢复。
3. 检查小程序类目与健康、运动记录功能是否匹配；页面不提供诊断或治疗结论。
4. 用正式 AppID 预览并测试至少一台 iOS 和一台 Android 真机。
5. 在开发者工具上传代码，登录公众平台填写版本说明、提交审核，通过后再发布。

## 当前边界

饮食页面当前使用本地模板演示“拍照—分析—确认—保存”的完整交互，不会调用 Qwen-VL 或其他收费图像接口。后续接入真实识别时，应在云函数中保存密钥，并在用户确认前明确图片是否上传、保存多久以及如何删除；不要把 API Key 写入小程序前端。

## 本地逻辑测试

在仓库根目录执行：

```powershell
npm test
```

测试覆盖训练轮换、连续打卡、健康分、重量建议、营养换算、输入范围、本地存储与备份白名单。
